Hyper Text Markup Language

HTML describes the structure of web pages semantically.

Markup Language
because we describe what meaning a piece of text on a web page has
by putting markers around them
or indicate them with a single marker.

We call those markers (HTML) tags. Tags make up an (HTML) element. They can be enclosing or self-closing.

<p>is enclosing</p>
<hr> is self-closing

Tags can have attributes in the opening tag. Those attributes can serve different purposes. They usually work like: key="value"

This `p` element has a title attribute.

Hyper Text
because we can reference documents outside of the current document.

The most important element in that regard is the anchor tag.

<a href="#">this is what you'd call a link</a>

Notice the `href` attribute. It stands for `hyper reference` and can address elements of this page or other documents. Or even elements on other documents!

He was tired of walking the steps at CERN, so he invented the Internet.

Meaning a way to write documents that made sense of their structure and
a way that computers could exchange those documents,

Broswers

Tim also wrote a browser.

A Browser can make sense of the HTML structure and render (or display) an HTML page to a user in a easy to look at and understand manner.

Just like this document you're looking at right now.

Tim called his browser WorldWideWeb, later renamed to Nexus because WWW was then used for the whole system of webpages.

Web browsers gained massive popularity with Mosaic, whose interface is still the basis for browser interfaces today.

Mosaic was discontinued, but most of the team then worked on the Netscape Navigator which later would become the basis for the first Firefox browser.

The 90ies became famous for the so called Browser Wars. Different browsers emerged and fought for marked share. They implemented elements and other parts differently and it was not nice to develop functioning cross-browser websites.

The wars are considered over with an understanding that open web standards are the goal browser vendors aspire to.

Web Standards

Speaking of standards. Who decides on new elements and such?

That would be the W3C.

They organise different working groups for web standards and work on specificatins that browser vendors then implement.

The HTML working group is called WHATWG.

The cool thing about the W3C is that ideas for new features come from individuals and companies and institutions alike.
If you have strong opinions and a lot of time, you can participate!

A recent example of this is the responsive images group.
They consist of frontend developers who were not satisfied with the drafts the W3C put out for new image elements.
So they proposed their own spec and it's now part of the standard!

From web page to web app

Web pages and web apps these days are more than just text documents. They come with sophisticated interfaces, dynamic content generated on servers talking to databases, individually cut for each user. And they look very shiny.

But in the end, no matter what backend language you use, you will generate HMTL that describes content and send it to your users.

Let's talk about CSS

Cascading Style Sheets

CSS is the styling langauge of the web. Where HTML gives content meaning and structure, CSS gives it good looks.

The `cascading` in the name comes from more advanced concepts that deal with hierarchy and specificity. But to get started we don't need to know about that.

Next

How do I CSS?

CSS syntax follows simple rules.

element {
  property: value;
}

That's it!

You can address elements with selectors. There is very good documentation of how selectors work on the Mozilla Developer Network.

How do I know what properties there are? You have to learn about them. It's slow and painful and I guess the reason why a lot of developers don't like writing CSS.

There is a workshop called CSS Classes in Berlin and they have some nice resources and workshop materials online.
